# Documentation

Use these files as the current documentation set:

- [Flow spec](../README.md): product contract and user-flow sequence diagrams.
- [Runbook](run.md): local startup, verification commands, and mainnet gates.
- [Manual test checklist](manual-test-checklist.md): required checks before calling work done.
- [Manual evidence log](manual-test-evidence.md): template for recording command output, screenshots, and manual judgments.
- [Task backlog](tasks): implementation tasks derived from the flow spec and current code gaps.

Do not add historical implementation notes here. If a doc says a feature passed, it must point to fresh evidence in `manual-test-evidence.md`.
